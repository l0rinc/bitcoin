# Issue 35298 Reproducers

These commands are formatted as readable multi-line shell commands, but each block is a single shell history item when pasted with the trailing backslashes intact.

Run the same command on a control commit and on this branch, changing `LABEL` each time. Keep `DATADIR`, `DBCACHE`, `MAXMEMPOOL`, `TXINDEX`, and hardware the same between runs. The commands write all raw samples plus parsed summaries under `issue-35298/runs/$LABEL/<timestamp>/`.

## Steady State

Use this on an already synced datadir. Set `DURATION=3600` for one hour or `DURATION=7200` for two hours.

```bash
DATADIR=/path/to/BitcoinData; \
BUILD_DIR="$PWD/build"; \
LABEL=patched-steady; \
RUN_BASE="$PWD/issue-35298/runs/$LABEL"; \
INTERVAL=10; \
DURATION=7200; \
DBCACHE=800; \
MAXMEMPOOL=100; \
TXINDEX=0; \
WAIT_UNTIL_SYNCED_BEFORE_TIMER=1; \
STOP_WHEN_SYNCED=0; \
REQUIRE_CHAINSTATE=1; \
export DATADIR BUILD_DIR RUN_BASE INTERVAL DURATION DBCACHE MAXMEMPOOL TXINDEX WAIT_UNTIL_SYNCED_BEFORE_TIMER STOP_WHEN_SYNCED REQUIRE_CHAINSTATE; \
issue-35298/monitor.sh && \
RUN_DIR="$(ls -td "$RUN_BASE"/* | head -n1)" && \
issue-35298/summarize_io.py "$RUN_DIR" --steady-state --fully-synced > "$RUN_DIR/io-summary.json" && \
issue-35298/parse_compactions.py "$RUN_DIR/debug.log" > "$RUN_DIR/compaction-waves.tsv" && \
issue-35298/chainstate_age.py "$DATADIR/chainstate" > "$RUN_DIR/chainstate-age-end.tsv" && \
printf 'run_dir=%s\n' "$RUN_DIR"
```

To check whether indexes amplify the issue, repeat the steady-state command with `TXINDEX=1` and a different `LABEL`, using the same commit and datadir after a clean shutdown.

## Reindex Chainstate

Use this on an existing datadir. `DURATION` is only a safety ceiling; the monitor stops once `getblockchaininfo` reports `initialblockdownload=false` and `blocks==headers`.

```bash
DATADIR=/path/to/BitcoinData; \
BUILD_DIR="$PWD/build"; \
LABEL=patched-reindex-chainstate; \
RUN_BASE="$PWD/issue-35298/runs/$LABEL"; \
INTERVAL=30; \
DURATION=604800; \
DBCACHE=800; \
MAXMEMPOOL=100; \
TXINDEX=0; \
EXTRA_BITCOIND_ARGS="-reindex-chainstate=1"; \
WAIT_UNTIL_SYNCED_BEFORE_TIMER=0; \
STOP_WHEN_SYNCED=1; \
REQUIRE_CHAINSTATE=1; \
export DATADIR BUILD_DIR RUN_BASE INTERVAL DURATION DBCACHE MAXMEMPOOL TXINDEX EXTRA_BITCOIND_ARGS WAIT_UNTIL_SYNCED_BEFORE_TIMER STOP_WHEN_SYNCED REQUIRE_CHAINSTATE; \
issue-35298/monitor.sh && \
RUN_DIR="$(ls -td "$RUN_BASE"/* | head -n1)" && \
issue-35298/summarize_io.py "$RUN_DIR" > "$RUN_DIR/io-summary.json" && \
issue-35298/parse_compactions.py "$RUN_DIR/debug.log" > "$RUN_DIR/compaction-waves.tsv" && \
issue-35298/chainstate_age.py "$DATADIR/chainstate" > "$RUN_DIR/chainstate-age-end.tsv" && \
printf 'run_dir=%s\n' "$RUN_DIR"
```

## Full IBD

Use an empty datadir. This command refuses to run if `blocks/` or `chainstate/` already exists.

```bash
DATADIR=/path/to/empty-ibd-datadir; \
BUILD_DIR="$PWD/build"; \
LABEL=patched-ibd; \
RUN_BASE="$PWD/issue-35298/runs/$LABEL"; \
INTERVAL=30; \
DURATION=1209600; \
DBCACHE=800; \
MAXMEMPOOL=100; \
TXINDEX=0; \
EXTRA_BITCOIND_ARGS=""; \
WAIT_UNTIL_SYNCED_BEFORE_TIMER=0; \
STOP_WHEN_SYNCED=1; \
REQUIRE_CHAINSTATE=0; \
test ! -e "$DATADIR/blocks" && \
test ! -e "$DATADIR/chainstate" && \
mkdir -p "$DATADIR" && \
export DATADIR BUILD_DIR RUN_BASE INTERVAL DURATION DBCACHE MAXMEMPOOL TXINDEX EXTRA_BITCOIND_ARGS WAIT_UNTIL_SYNCED_BEFORE_TIMER STOP_WHEN_SYNCED REQUIRE_CHAINSTATE; \
issue-35298/monitor.sh && \
RUN_DIR="$(ls -td "$RUN_BASE"/* | head -n1)" && \
issue-35298/summarize_io.py "$RUN_DIR" > "$RUN_DIR/io-summary.json" && \
issue-35298/parse_compactions.py "$RUN_DIR/debug.log" > "$RUN_DIR/compaction-waves.tsv" && \
issue-35298/chainstate_age.py "$DATADIR/chainstate" > "$RUN_DIR/chainstate-age-end.tsv" && \
printf 'run_dir=%s\n' "$RUN_DIR"
```

## Useful Outputs

`io-summary.json` contains process and block-device read/write deltas.

`compaction-waves.tsv` summarizes LevelDB compaction waves, generated bytes by output level, and nearby flush/cache context.

`chainstate-age-end.tsv` reports what percentage of the final chainstate files were rewritten recently.

`start-command.sh`, `run.env`, `git-head.txt`, `git-status.txt`, `debug.log`, `proc_io.tsv`, `diskstats.tsv`, `chainstate.tsv`, and `blockchaininfo.jsonl` are kept in the same run directory for plotting and audit.
